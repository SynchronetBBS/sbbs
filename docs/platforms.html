★ = current release  •  BSD = FreeBSD, NetBSD & OpenBSD  •  Vista/7 = NT 6.0–6.1 (incl. Server 2008 R2)  •  8–11 = NT 6.2–10 (incl. Server 2012–2022)  •  Yes* = source builds only, no official binaries  •  Solaris 8+ and QNX supported from v3.11; source builds only [5]

[1]
DOS/OS2 emulation only (NTVDM on Windows, DOSEMU on Linux) — no TCP/IP; all BBS services require modem/FOSSIL hardware. NTVDM is present in 32-bit editions of Windows through Windows 10.
[2]
No FOSSIL/interrupt driver — external programs and door games not supported
[3]
Windows 95 (original) and Windows NT 3.x not supported due to missing GetDiskFreeSpaceEx; Windows 95 OSR2+ or Windows NT 4+ required
[4]
GUI (sbbsctrl) requires OLEAUT32.DLL; obtain by upgrading Internet Explorer
[5]
FreeBSD, NetBSD, and OpenBSD were added incrementally across v3.10h–3.12; some variants may lack full support in earlier sub-versions of this range
[6]
No configuration utilities (SCFG) in early Unix ports
[7]
No JavaScript support in early Unix ports
[a]
v3.20 is the last release to officially support Windows Vista and 7. In v3.21, scfg.exe and echocfg.exe fail on Windows 7 due to a new dependency on GetOverlappedResultEx (absent from the Windows 7 kernel); patched utility builds were released separately.
[b]
Windows XP not supported — library dependencies in v3.20 require Windows Vista or later
[c]
Windows XP compatibility issues documented (e.g. missing inet_ntop); XP support was degrading prior to v3.20
[d]
Native DOS program; v2.30 also had a native OS/2 port. v3.00 and later are native Win32 / Unix programs.

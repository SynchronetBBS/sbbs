[1.1]  Charisma stat has no effect
-------------------------------------------------------------------------------
struct pc carries a Charisma attribute that can be set by class definitions
(mclass.c), items (mitems.c), and spells (mspells.c), and is shown in stat
displays.  It is never read in fight.c or empire.c, so it contributes nothing
to combat or empire outcomes.  Noted in progress.log as "charisma is useless!"

Options: give it a mechanical role (e.g. NPC loyalty, trading prices, odds of
seeing wandering NPCs, recruitment cost), or remove it from the data files and
display if there is no intention to use it.

[1.2]  Clean up mstrings.h GNUmakefile dependencies
-------------------------------------------------------------------------------
The mstrings.h target in src/GNUmakefile and the native-devkit dependency in
data/GNUmakefile both grew organically to solve the chicken-and-egg problem of
langcomp needing to exist before mstrings.h can be generated.  The current
approach works but could be cleaner -- the NATIVEEXEFILE computation and the
filter-out workaround for shared langcomp objects are worth revisiting.

[2.1]  qtest script mode: untestable scripting commands
-------------------------------------------------------------------------------
The qtest -s state summary (printed to stderr on exit) covers GFlags, HFlags,
PFlags, DFlags, TFlags, Gold, MineLevel, QuestsKnown, QuestsDone, and per-
member Name/HP/MaxHP/SP/MaxSP/XP/Level.  Several scripting commands change
state that is not included in this summary and therefore cannot be verified
beyond "exits with code 0":

Heal (HP restore)
        RunEvent() resets every present member's HP to MaxHP unconditionally
        at the end of every event (quests.c, after the command loop).  The
        Heal command also sets HP=MaxHP, so there is no observable difference
        between an event that calls Heal and one that does not.  A Heal HP
        test can only assert exit code 0.

GivePoints
        PClan.Points is not included in the state summary.

GiveFight
        PClan.FightsLeft is not included in the state summary.

GiveFollowers
        PClan.Empire.Army.Followers is not included in the state summary.

DoneQuest -> QuestsDone bit
        In qtest script mode, RunEvent() is called directly.  Setting
        QuestsDone is the caller's responsibility (Quests_GoQuest in the
        real game does this after RunEvent returns true).  qtest's main()
        does not set the bit when RunEvent returns true, so DoneQuest never
        produces a QuestsDone=N line in the state summary.  The command can
        still be tested for correct exit and stdout output
        ("Quest successfully completed!").

Options to improve coverage: extend the state summary in qtest.c to include
Points, FightsLeft, Followers, and the RunEvent return value; or add a
--dump-all flag that prints the full PClan struct.

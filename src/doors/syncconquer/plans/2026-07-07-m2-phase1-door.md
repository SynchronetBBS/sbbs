# SyncConquer M2: Phase-1 Door (syncalert) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps
> use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Red Alert skirmish playable over SyncTERM: the engine's backend
seam wired to libtermgfx (frames out via sixel/JXL/PPM/text tiers with
AIMD pacing; keyboard+SGR-mouse in; audio out), launched as a Synchronet
door.

**Architecture:** Engine `main()` untouched. The door lives in the three
backend translation units plus a door-core module they call:
`Video_Render_Frame()` = present hook (frames out),
`WWKeyboardClass::Fill_Buffer_From_System()` = input pump (socket bytes
in), soundio = audio out. All engine-seam facts are execution-verified in
`plans/m2-proto/FACTS.md`; the working prototype diff is
`plans/m2-proto/keystone.patch`; proof: `plans/m2-proto/menu-headless.png`.

**Tech Stack:** C (door core, matching termgfx/syncduke style), C++ (the
three engine-side shims), libtermgfx (../termgfx), CMake, xpdev.
